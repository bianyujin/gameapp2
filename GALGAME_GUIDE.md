### 3. 如何过滤敏感内容？

#### 方法一：使用"是否公开"字段

在Notion表格中添加 `是否公开` 列（Checkbox类型）：

| 游戏名 | 类型 | 是否公开 | 隐藏备注 |
|--------|------|----------|----------|
| AI女友 | 纯爱 | ✅ 勾选 | - |
| 敏感游戏A | 成人 | ❌ 不勾选 | 内部测试 |
| 普通游戏B | 恋爱 | ✅ 勾选 | - |

**效果**：
- ✅ 勾选 → 所有用户可见
- ❌ 不勾选 → 仅管理员可见

#### 方法二：使用"隐藏备注"字段

在Notion表格中添加 `隐藏备注` 列（Text类型）：

- 敏感信息写在"隐藏备注"列
- 应用会自动过滤，用户看不到
- 只有管理员登录后才能看到

#### 方法三：创建两个数据库

```
数据库1：公开游戏（用户可见）
数据库2：私密游戏（仅你可见）
```

### 4. 已有Notion表格如何使用？

**直接使用！** 无需重新创建。

#### 步骤：

1. **连接Integration**
   - 打开你的Notion表格
   - 点击右上角 `...` → `Add connections`
   - 选择你创建的Integration

2. **获取数据库ID**
   - 从URL中复制：`notion.so/workspace/DATABASE_ID?v=xxx`
   - 粘贴到应用设置中

3. **字段映射**
   
   应用会自动识别以下字段名（支持中英文）：

   | 应用字段 | Notion字段名（任选其一） |
   |----------|--------------------------|
   | 游戏名 | 标题、游戏名、Name、Title |
   | 图标 | 图标、Icon |
   | 类型 | 类型、分类、Category、Type |
   | 评分 | 评分、Rating、Score |
   | 社团 | 社团、开发商、Developer |
   | 评价 | 评价、Review |
   | 介绍 | 介绍、描述、Description |
   | 是否公开 | 是否公开、公开、isPublic |
   | 隐藏备注 | 隐藏备注、私密备注、HiddenNote |

## 🚀 快速开始

### 第一步：创建Integration（2分钟）

1. 访问 https://www.notion.so/my-integrations
2. 点击 `+ New integration`
3. 填写名称，选择工作区
4. 复制Token（以 `secret_` 开头）

### 第二步：连接表格（1分钟）

1. 打开你的Notion表格
2. 点击 `...` → `Add connections`
3. 选择刚创建的Integration
4. 复制数据库ID

### 第三步：配置应用（1分钟）

1. 打开应用 → 个人中心
2. 点击 云端设置
3. 选择 Notion
4. 粘贴Token和数据库ID
5. 点击"从云端获取"

**完成！** 以后只需在Notion中编辑，应用自动同步。

## 🔒 安全机制

### 三层过滤

```
Notion数据
    ↓
第一层：是否公开字段过滤
    ↓
第二层：隐藏备注字段过滤
    ↓
第三层：管理员权限验证
    ↓
展示给用户
```

### 敏感内容处理

| 内容类型 | 处理方式 |
|----------|----------|
| 不公开的游戏 | 用户完全看不到 |
| 隐藏备注字段 | 自动过滤，不显示 |
| 管理员专属内容 | 需登录验证 |

## ❓ 常见问题

### Q: 用户能看到我的Token吗？
A: 不会。Token存储在本地，不会暴露给用户。

### Q: 同步有延迟吗？
A: 通常几秒内完成。可在Notion中编辑后，在应用中点击"从云端获取"。

### Q: 可以多人管理吗？
A: 可以！Notion支持多人协作，团队成员都可以编辑数据。

### Q: 数据会丢失吗？
A: Notion有版本历史，可以随时恢复。建议定期导出备份。

### Q: 平台会审核内容吗？
A: 
- **Notion**：宽松，一般不审核
- **应用**：数据在本地，平台无法审核
- **用户**：只能看到公开内容
