# 📁 文件分类说明

---

## ✅ 需要部署到 GitHub Pages（用户访问用）

| 文件/文件夹 | 说明 |
|------------|------|
| `index.html` | 主 APP 入口 |
| `css/` | 样式文件 |
| `js/` | 脚本文件 |
| `icons/` | APP 图标 |
| `manifest.json` | PWA 配置 |
| `sw.js` | Service Worker |

---

## ⚠️ 需要上传到 GitHub Releases（数据和配置）

| 文件 | 说明 |
|------|------|
| `games.json` | 游戏数据（用一键发布工具生成） |
| `config.json` | APP 配置（用一键发布工具生成） |

---

## ❌ 不需要部署（仅本地使用）

| 文件 | 说明 |
|------|------|
| `一键发布工具.html` | 你本地处理数据用 |
| `upload-tool.html` | 旧工具，不用管 |
| `firebase-config-tool.html` | 旧工具，不用管 |
| `data-publish-tool.html` | 旧工具，不用管 |
| `*.md` | 各种文档，不用部署 |
| `firebase-config-example.json` | 示例文件，不用管 |
| `FIREBASE_*.md` | 旧文档，不用管 |
| `GITHUB_PAGES_*.md` | 旧文档，不用管 |

---

## 🚀 部署步骤

### 第一步：部署 APP 到 GitHub Pages

1. 在 GitHub 创建仓库（比如叫 `game-app`）
2. 上传以下文件到仓库：
   - `index.html`
   - `css/` 文件夹
   - `js/` 文件夹
   - `icons/` 文件夹
   - `manifest.json`
   - `sw.js`
3. 打开仓库 Settings → Pages
4. Source 选择 `main` 分支，点击 Save
5. 等几分钟，APP 就上线了！

---

### 第二步：上传数据到 GitHub Releases

1. 用 `一键发布工具.html` 生成 `games.json` 和 `config.json`
2. 打开仓库 → Releases → Draft a new release
3. Tag version 填：`v1.0.0`（以后每次更新加版本号）
4. 把 `games.json` 和 `config.json` 拖进去
5. 点击 Publish release
6. 复制两个文件的下载链接

---

### 第三步：APP 首次配置

1. 打开部署好的 APP
2. 进入"我的"页面
3. 连点版本号 5 次 → 登录管理员
4. 在管理员面板的"设置"里填 `config.json` 的下载链接
5. 点击同步 → 完成！

---

## 📋 日常更新

1. Notion 导出 CSV
2. 用 `一键发布工具.html` 处理
3. 下载新的 `games.json` 和 `config.json`
4. 在 GitHub Releases 发布新版本，替换旧文件
5. 用户点击同步 → 拿到新数据！

---

## 🎯 总结

| 项目 | 位置 | 频率 |
|------|------|------|
| APP（index.html等） | GitHub Pages | 很少更新 |
| 数据（games.json） | GitHub Releases | 经常更新 |
| 配置（config.json） | GitHub Releases | 偶尔更新 |
| 工具（一键发布工具.html） | 本地电脑 | 每次更新数据用 |
