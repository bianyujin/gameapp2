# GAMEACG GalGame 资讯平台 - 产品需求文档

## 1. 产品概述

一款面向国内用户的 GalGame（视觉小说/美少女游戏）资讯整合移动端应用，整合游戏信息、攻略、评分评价、下载指引等内容。核心目标：**零成本、无需VPN、自动化更新（改Notion即生效）、无需签名即可安装、UI美观、内容完全封装在App内**。

- **目标用户**: 国内 GalGame 爱好者，习惯用手机浏览游戏资讯
- **核心价值**: 管理员在 Notion 网页端编辑表格 → 自动同步 → 用户在 App 内看到最新数据，全程无需手动导出/打包

## 2. 核心功能

### 2.1 用户角色

| 角色 | 获取方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 打开App即用 | 浏览游戏列表、搜索、查看详情（含所有自定义字段）、收藏、评分评论、**复制链接** |
| 管理员 | 版本号连点5次输入密码 | 编辑/添加/删除游戏数据、管理字段顺序、触发云端同步 |

### 2.2 功能模块

1. **首页**: 随机壁纸轮播、分类快捷入口、热门/最新游戏卡片列表
2. **数据页**: 全部游戏展示、搜索、筛选（分类/评分）、排序、分页
3. **详情弹窗**: 游戏基本信息 + 自定义字段（下载链接、攻略、评分细则等）+ **每个字段一键复制**
4. **个人中心**: 收藏列表、浏览历史、深色模式切换、云端数据同步、关于
5. **管理后台**（隐藏入口）: 数据增删改、字段排序、JSON/CSV导入导出、手动触发同步

### 2.3 页面功能清单

| 页面 | 模块 | 功能描述 |
|------|------|----------|
| 首页 | 顶部引导横幅 | 首次使用提示同步数据，可关闭 |

## 3. 核心流程

### 用户使用流程
```
打开App → 自动检查更新(每小时) → 有新数据提示同步 → 浏览/搜索游戏 → 点击查看详情 → 复制需要的链接或信息到外部浏览器使用 → 可收藏/评价
```

### 管理员更新流程（全自动化）
```
管理员在 Notion 网页端编辑表格
        ↓ (自动化)
GitHub Actions 定时任务(每30分钟) 或 手动触发
        ↓
调用 Notion API 读取数据库 → 转换为 games.json 格式
        ↓
自动 Commit + Push 到仓库 → 创建 GitHub Release
        ↓
用户打开 App → 检测到新版本数据 → 一键同步 → 看到最新内容
```

### 关键安全规则
- **所有外部链接（百度网盘/迅雷/UC/攻略/DL站等）仅提供复制功能**
- **App 内不允许任何链接被点击打开**（防止追踪/暴露来源）
- **用户无法获取 Notion 原始链接或 API Token**
- **Notion 内嵌页面功能移除**（数据完全通过 JSON 同步到 App 内）

## 4. 数据同步策略（优先级从高到低）

```
1. GitHub CDN (jsDelivr) — 主力数据源，由GitHub Actions自动生成
2. Firebase Realtime Database — 备用数据源（手动同步）
3. LocalStorage 缓存 — 离线兜底
```

## 5. 用户界面设计

### 5.1 设计风格
- **主色调**: 深靛蓝/紫色渐变 (#1e1b4b → #6366f1)，二次元风格
- **辅助色**: 琥珀色(#f59e0b)用于警告/管理员标识
- **背景**: 深色系 (#0f172a 主背景, #1e293b 卡片背景)
- **文字**: 主文字 #e2e8f0，次要文字 #94a3b8
- **按钮**: 圆角胶囊状，主操作 indigo 渐变，次操作灰色
- **字体**: 系统默认中文字体栈
- **布局**: 移动端优先，底部三Tab导航（首页/数据/我的），卡片式设计
- **图标**: Emoji 图标（轻量无需图标库）

### 5.2 链接交互规范

| 字段类型 | 交互方式 | 说明 |
|----------|----------|------|
| 下载链接（百度/迅雷/UC等） | 仅显示文本 + 复制按钮 | 不可点击，不可在App内打开 |
| 攻略链接 | 仅显示文本 + 复制按钮 | 同上 |
| DL号/社团/作者 | 仅显示文本 + 复制按钮 | 同上 |
| 普通文本字段 | 仅显示文本 + 复制按钮 | 同上 |
| 封面图片URL | 可预览（可选） | 不自动跳转 |

### 5.3 响应式设计
- **主要目标**: 手机竖屏体验（375px-430px宽度）
- **触摸优化**: 最小点击区域 44px，复制按钮易于触控

## 6. 非功能性需求

### 6.1 数据源要求（国内可用性）

| 服务 | 国内访问 | 免费额度 | 用途 |
|------|----------|----------|------|
| GitHub + jsDelivr CDN | ✅ 大部分区域可访问 | 无限 | 数据CDN分发（主力，由Actions自动推送） |
| Gitee | ✅ 无需VPN | 公开仓库无限 | 可选的国内镜像备选 |
| Firebase (asia-southeast1) | ✅ 无需VPN | 免费1GB/月 | 实时数据库备用 |
| Notion API | ⚠️ 可能需要VPN | 免费100万次/月 | 仅服务器端(GitHub Actions)调用，用户不直接访问 |

### 6.2 安装方式
- **PWA**: 浏览器"添加到主屏幕"，无需签名，iOS/Android通用
- **Capacitor APK**: Android签名内置，无需手动配置签名文件
- **在线访问**: 直接浏览器打开URL即可使用

### 6.3 更新机制
- **数据热更新（全自动）**: 管理员改 Notion → Actions 自动同步 → 用户重启App或点同步即看到新数据
- **App版本更新**: 通过网盘分享APK，config.json 中配置更新链接
- **无需重新安装App** 即可获取最新游戏数据
