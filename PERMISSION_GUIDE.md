## 🔑 管理员功能

### 首次使用

1. 进入 **个人中心** → **管理员** → **进入**
2. 系统会提示设置管理员密码
3. 输入密码并保存

### 管理员面板功能

| 功能 | 说明 |
|------|------|
| 添加游戏 | 添加新的游戏数据 |
| 导出数据 | 备份所有数据（JSON格式） |
| 批量导入 | 从JSON文件导入数据 |
| 评论管理 | 查看/删除用户评论 |
| 密码设置 | 修改管理员密码 |

### 数据管理方式

**方式一：在应用内管理**
- 登录管理员后，右下角会出现 **+** 按钮
- 点击可添加新游戏
- 在数据表格中可编辑/删除游戏

**方式二：通过Notion管理（推荐）**
- 在Notion网页中直接编辑数据
- 应用会自动同步最新数据
- 更方便批量操作

## ☁️ 云端同步配置

### Firebase方案（推荐）

**为什么推荐Firebase？**
- ✅ 完全免费（1GB存储 + 10GB流量/月）
- ✅ 支持权限规则控制
- ✅ 实时同步
- ✅ 国内访问稳定（使用亚洲节点）

**配置步骤：**

1. 创建Firebase项目
2. 创建实时数据库
3. 设置安全规则：

```json
{
  "rules": {
    "games": {
      ".read": true,
      ".write": false
    },
    "reviews": {
      ".read": true,
      ".write": true
    }
  }
}
```

这个规则实现：
- 游戏数据：所有人可读，无人可写（只能通过控制台修改）
- 评论数据：所有人可读可写

4. 在应用中配置Firebase

## 🔒 安全建议

### 管理员密码

- 使用强密码（8位以上，包含数字和字母）
- 不要使用常见密码
- 定期更换密码

### Firebase安全

生产环境建议添加认证：

```json
{
  "rules": {
    "games": {
      ".read": true,
      ".write": "auth != null && root.child('admins').child(auth.uid).exists()"
    },
    "reviews": {
      ".read": true,
      "$reviewId": {
        ".write": "auth != null && (!data.exists() || data.child('userId').val() === auth.uid)"
      }
    }
  }
}
```

## ❓ 常见问题

### Q: 用户可以修改游戏数据吗？
A: 不可以。普通用户只能查看游戏数据，无法修改。

### Q: 如何批量添加游戏？
A: 两种方式：
1. 管理员面板 → 批量导入（JSON文件）
2. 在Notion中直接编辑，然后同步

### Q: 用户评论会被其他人删除吗？
A: 不会。用户只能删除自己的评论，管理员可以删除所有评论。

### Q: 数据会丢失吗？
A: 
- 本地数据：清除浏览器数据会丢失
- 云端数据：永久保存在Firebase/Notion
- 建议：定期导出备份

### Q: 如何防止管理员密码被破解？
A: 
- 使用强密码
- 密码存储在本地，不会上传到云端
- 打包APK后更安全（代码混淆）
